## Content Checks

- [ ] Business background clearly states the churn problem.
- [ ] Problem definition explains why churn matters for recurring revenue.
- [ ] Dataset overview matches the prepared feature set.
- [ ] EDA findings are grounded in the existing reports.
- [ ] Modeling approach matches the selected features and explainability work.
- [ ] Model results mention the key drivers without overstating accuracy.
- [ ] Churn drivers are tied to device, usage, and value signals.
- [ ] Business proposal includes clear initiatives and prioritization.
- [ ] Impact estimation explains assumptions rather than inventing precision.
- [ ] Risks are explicitly stated.
- [ ] Next steps are operational and measurable.

## Presentation Checks

- [ ] Slide flow matches the required 12-slide structure.
- [ ] Speaker notes are prepared for each major section.
- [ ] Claims are consistent across all reports.
- [ ] No new analysis contradicts the model and EDA outputs.
- [ ] Final narrative uses business language, not only technical language.

## Export and Delivery Checks

- [ ] Final files are saved in `docs/final-submission/`.
- [ ] All report paths are correct and readable.
- [ ] The final package is ready for handoff.
